AI Text-to-Video Orchestration Platform

把长文本内容，变成可追踪、可调度、可批量执行的视频生产流水线。

JianDou 面向短剧与叙事型内容生产，把 TXT 输入、文本分析、创意提示词、视频生成、任务追踪和结果回传收束为一条统一任务链路。 它不是单次提示词试错工具，而是更接近生产系统的 AI 视频编排平台。

Pipeline

从输入文本到输出视频，不靠手工拼流程

Step 1

采集输入

上传 TXT 或直接输入文本，同时确定比例、时长、文案风格和目标模型。

Step 2

分析与规划

先做文本理解，再生成创意提示词与镜头规划，减少直接调用视频模型的盲打成本。

Step 3

异步渲染

任务被投递到队列，由 Worker 执行生成任务，前台只负责追踪和管理，不阻塞页面交互。

Step 4

结果沉淀

生成产物、过程日志、状态变化和模型调用结果统一落库和落盘，便于复用、审计和排查。

Architecture

前后端分离 + Worker 解耦，让视频生成真正进入任务系统

Core Stack

apps/web

Vue 3 + Vite 前端，承担任务创建、列表、详情、日志和运行态可视化。

apps/api

FastAPI API 网关，对外暴露 `/api/v2`，管理任务、生成、上传和管理端接口。

apps/worker

消费 Redis 队列，执行分析、规划、渲染等异步阶段，把高耗时任务从请求线程剥离。

packages/*

AI 编排、任务流水线、数据库模型、存储封装与公共 schema 独立沉淀，便于后续扩展。

Data Plane

MySQL + Redis + Local Storage

任务元数据写入 MySQL，队列交给 Redis，上传文件和生成结果落在本地存储目录。

Config

app.toml 与环境变量覆盖

支持通过 `config/app.toml` 和 `JIANDOU_*` 环境变量管理模型 Key、数据库与运行参数。

FAQ

部署、使用与项目边界

JianDou 当前更适合什么团队？

更适合已经在做短剧、剧情预告、章节拆解或 AI 视频试产的团队，需要的是任务系统，而不是单次生成玩具。

当前部署方式是什么？

文档支持 Docker Compose 和本地直跑两种模式。项目说明里同时提示：当前阶段更建议本地开发模式启动，稳定后再发 Docker 版本。

是否支持任务级排错？

支持。任务详情、trace、状态历史、模型调用记录、日志和结果列表都能帮助定位问题与回溯过程。

是否已经做成完整 SaaS？

还没有。当前更准确的定位是一个面向内容生产场景的 AI 视频生成平台工程基础盘，已经具备持续扩展能力。
